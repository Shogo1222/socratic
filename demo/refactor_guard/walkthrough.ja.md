# Walkthrough: 「ただのリファクタリング」を`$socratic`でレビューする

リファクタリングPRに対するセッションの実況です。振る舞い差が見つかったとき、何が表示され、何を聞かれ、答えるとどうなるかを示します。

## 1. 状態

PRの説明は「可読性のためのリファクタリング。振る舞いの変更なし」。差分は`renew()`から`_is_active()`ヘルパーを抽出しただけに見えます。実際には、抽出時に比較が`<=`から`<`へ変わっています。

## 2. 実行

```text
> $socratic このリファクタリングPRをレビューして
```

## 3. ターミナル(比較フェーズ)

```text
Socratic: 振る舞い維持を主張する変更のため、Refactor Guardとして進めます。

  BaseとHeadをDisposable Snapshotとして展開...
  - Baseの重要な振る舞いを観測するProbeを3件生成
    (出力と例外だけを観測。内部構造には触れません)
  - 同じProbeをBaseとHeadの両方へ実行

  Probe                         Base   Head
  終了日よりかなり前の更新       pass   pass
  終了日よりかなり後の拒否       pass   pass
  終了日当日の更新               pass   FAIL   <- Behavior Diff
```

## 4. 質問(構造化UI)

Baseは仕様ではなく、変更前に観測された事実として提示されます。

```text
┌ 質問 1/1 ── 振る舞い差 ───────────────────────────────────┐
│ このリファクタリングで、期限境界の振る舞いが変わっています │
│                                                            │
│   変更前: 契約終了日当日の更新は成功                       │
│   変更後: ExpiredSubscriptionError                         │
│                                                            │
│ この変更は意図したものですか?                              │
│                                                            │
│   1. 意図した変更                                          │
│      新しい期待値として記録し、テストで固定します          │
│ ▸ 2. 意図していない                                        │
│      リファクタリングが持ち込んだ回帰として報告します      │
│   3. その他(自由入力)                                      │
└────────────────────────────────────────────────────────────┘
あなた: 2. 意図していない
```

## 5. 結果(正準の4ブロック)

```text
Socratic Review

Review This:
  (人間の判断は残っていません — 振る舞い差は「意図していない」と確定)

We Verified:
  ✓ 終了日前の更新と終了日後の拒否は維持 (Probe 2件、Base/Head両方でpass)
  ✓ 期限境界の回帰は、Disposable環境で提案・証明済みのProbeが検知 (Strong Catch)
  ✓ 今回のReview-only実行中、Working Treeは不変

Still at Risk:
  △ 境界を固定する提案Probeは未適用
    次の同種の回帰は、適用するまで自動検知されません

Copy-ready Comments:
  1 comment for subscription/renew.py:12
```

## 6. コメント候補(そのまま貼れる形)

```markdown
このリファクタリングでは、期限切れ判定の境界条件が変わっているようです。

契約終了日と実行日が同じケースについて、変更前は更新できますが、
変更後は拒否されます(ExpiredSubscriptionError)。

振る舞いを維持するリファクタリングとのことなので、意図しない変更の
可能性があります。抽出された_is_activeの比較を`<=`へ戻すか、境界の
変更が意図的であれば仕様として明記いただけますか?
```

投稿するか、編集するか、破棄するかはあなたが決めます。Socraticは投稿しません。

## 7. 後処理

```text
証明済みProbeの扱い: テストを適用する / Patchを出力する / 破棄する
あなた: Patchを出力する
  -> 境界Probeのテスト専用Patchが表示されます (Working Treeは不変)

実行時の成果物: 保存しない(既定) / ローカルに保存 / Markdownとして出力
あなた: 保存しない
```

## この題材を自分で動かす

```bash
python3 -m demo.refactor_guard.run_demo
```

同じProbe行列と「意図した変更ですか?」までを10秒で再現します。
